# HTTP коды ошибок и обработка

## Коды статусов

### Успешные

| Код | Значение | Когда |
|---|---|---|
| 200 | OK | Запрос выполнен, данные в теле ответа |
| 201 | Created | Ресурс создан (карточка, склад, кампания) |
| 204 | No Content | Успешно, но данных нет (нет новых отчётов, пустой результат) |

### Клиентские ошибки (не ретраить, кроме 429)

| Код | Значение | Причина | Что делать |
|---|---|---|---|
| 400 | Bad Request | Ошибка в синтаксисе, параметрах, JSON | Проверить URL, параметры, тело запроса |
| 401 | Unauthorized | Неправильный / отсутствующий / истёкший токен | Проверить `Authorization: Bearer`, выпустить новый токен |
| 403 | Forbidden | Нет нужного scope в токене | Создать новый токен с нужной категорией |
| 404 | Not Found | nmID/orderId/warehouseId не существует | Проверить ID, ресурс мог быть удалён |
| 405 | Method Not Allowed | Метод не поддерживается этим путём — частый маркер устаревших путей | Проверить HTTP-метод и актуальность пути (см. `15-advertising.md`) |
| 409 | Conflict | Дубликат, версия устарела | Повторить с актуальными данными |
| 429 | Too Many Requests | Превышен rate limit | **Ретраить** — см. `01-rate-limits-retry.md` |

### Серверные ошибки (ретраить с backoff)

| Код | Значение | Причина | Что делать |
|---|---|---|---|
| 500 | Internal Server Error | Ошибка на стороне WB | Повторить через 5-10 сек с backoff |
| 502 | Bad Gateway | Ошибка в цепочке обработки | Повторить через 5-10 сек |
| 503 | Service Unavailable | Техническое обслуживание | Повторить через 30-60 сек |

## Формат тела ошибки

```json
{
  "title": "Error title",
  "detail": "Detailed error message",
  "status": 401,
  "statusText": "Unauthorized",
  "timestamp": "2024-06-20T10:30:45.123Z",
  "errors": [
    {
      "message": "Invalid token",
      "code": "INVALID_TOKEN"
    }
  ]
}
```

Не все поля гарантированно присутствуют. Всегда проверять `status` + `detail`.

## Типичные проблемы и решения

### «401 Unauthorized» при первом запросе
- Токен не передан или передан без `Bearer`
- Токен истёк (180 дней) — создать новый
- Токен для sandbox, а запрос к prod API (или наоборот)

### «403 Forbidden» на конкретный эндпоинт
- При создании токена не выбрана нужная категория (scope)
- Например: для рекламы — scope **Promotion**; для финотчёта зависит от API — у deprecated `reportDetailByPeriod` scope **Statistics**, у актуального Finance API — scope **Finance**
- Решение: создать новый токен с правильными scopes

### «429 Too Many Requests» постоянно
- Слишком частые запросы к одному семейству
- Для Statistics API: `reportDetailByPeriod` — строго 1 запрос в минуту
- Решение: увеличить интервал, использовать backoff, см. `01-rate-limits-retry.md`

### «409 Conflict» при обновлении
- Попытка обновить ресурс, который уже изменён
- В Marketplace API: 409 = 5 обычных запросов (по лимитам), для DBS = 10
- Решение: перечитать ресурс и повторить

### Пустой ответ 204
- Это **не ошибка** — просто нет данных за запрошенный период
- Часто при запросе новых сборочных заданий, когда заказов нет
